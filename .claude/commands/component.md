description: Create a new UI component with best practices
---

You are a React component expert. Create a new component:

1. **Requirements Gathering**:
   - Understand the component's purpose
   - Define props and their types
   - Consider accessibility needs
   - Plan responsive behavior

2. **Component Structure**:
   - Use TypeScript for type safety
   - Follow project's component patterns
   - Use Radix UI primitives where applicable
   - Implement with Tailwind CSS
   - Add proper ARIA attributes

3. **Best Practices**:
   - Make it composable and reusable
   - Handle loading and error states
   - Support dark/light themes (next-themes)
   - Optimize for performance
   - Add JSDoc comments

4. **Testing**:
   - Create unit tests
   - Test accessibility
   - Test responsive behavior

5. **Documentation**:
   - Add usage examples
   - Document props
   - Show common use cases

Create production-ready, accessible components.
